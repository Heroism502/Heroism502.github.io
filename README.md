# Academic Homepage

Static GitHub Pages homepage.

## Deploy to GitHub Pages for Free

Use the GitHub account `Heroism502` and create or rename the repository to:

```text
Heroism502.github.io
```

Put these files in the repository root:

```text
index.html
styles.css
script.js
pic.png
robots.txt
.nojekyll
README.md
.gitignore
```

After deployment, the free GitHub Pages address will be:

```text
https://Heroism502.github.io/
```

## GitHub Setup Flow

1. On GitHub, sign in to `Heroism502`.
2. Create a new public repository named `Heroism502.github.io`.
3. Upload or push the website files in this folder to the repository root.
4. Open repository `Settings` -> `Pages`.
5. Set `Source` to `Deploy from a branch`.
6. Choose branch `main` and folder `/root`.
7. Save and wait for GitHub Pages to deploy.
8. Visit `https://Heroism502.github.io/`.

## Preview Access

The current page is in owner-preview mode. This is a static-site visibility gate, plus `robots.txt` and `noindex` metadata. It prevents casual web viewing and indexing, but it is not a replacement for a private repository or server-side authentication.

To change the access code, calculate a SHA-256 hash of the new code and replace `ACCESS_HASH` in `script.js`.
